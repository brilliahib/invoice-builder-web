# Glossary

## Terms

| Term | Definition |
|------|-----------|
| **Company** | The business entity a user belongs to. Used as the invoice identity source. |
| **Invoice** | A document issued by the company to a client for services or goods rendered. |
| **Line Item** | A single product or service entry within an invoice (quantity × unit price). |
| **Company Scope** | The principle that all data is isolated per company. No cross-company access is allowed. |
| **Draft** | An invoice status indicating the invoice is not yet finalized or sent. |
| **Published** | An invoice status indicating the invoice has been finalized and sent to the client. |
| **PDF Export** | The action of generating a downloadable PDF document from an invoice record. |
| **Signatory** | The person from the company who signs or authorizes the invoice. |
| **Use Case** | An application-layer class that encapsulates a single business operation. |
| **Repository** | An abstraction over the data persistence layer (Prisma implementation). |
| **DTO** | Data Transfer Object — a typed object used to move data between layers. |
| **RLS** | Row Level Security — Supabase/PostgreSQL feature to enforce data access policies. |
| **React Query** | TanStack Query — library used for all server state management in this application. |
| **Atomic Design** | Frontend component hierarchy: Atoms → Molecules → Organisms → Templates → Pages. |
